# GitHub Project Issue Workflow

Each GitHub issue represents one small learning task.

## Issue structure

- Goal: what should be achieved.
- Checklist: practical steps to complete.
- Notes: important commands, mistakes, and fixes.
- Public result: README, screenshots, summary, or published repository.

## Board status

- Backlog: planned task.
- In progress: currently active task.
- Done: practical work completed.
- Published: public result prepared.
