# DevOps Learning Workflow

This repository is used as a public learning tracker for DevOps practice.

## Workflow

1. Create learning issues in GitHub Project.
2. Move issues through Backlog, In progress, Done, and Published.
3. Save useful commands, notes, and public summaries.
4. Link practical work to repositories, commits, or README files.

## Purpose

The goal is to keep DevOps learning structured, visible, and portfolio-ready.
